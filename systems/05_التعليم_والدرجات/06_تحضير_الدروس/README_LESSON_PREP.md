# تحضير الدروس (Subsystem 06)
## `DDL_LESSON_PREP.sql`

هذا النظام الفرعي يضمن أن التحضير ليس عملًا عشوائيًا، بل خطة موثقة قابلة للمتابعة والاعتماد.

## شرح مبسط
- المعلم يسجل تحضير الدرس قبل التنفيذ.
- المشرف أو الإدارة تراجع التحضير وتحدد حالة الاعتماد.
- يسهل استخراج تقارير يومية/أسبوعية/شهرية عن جاهزية التدريس.

## الجدول الأساسي

| الجدول | وظيفته |
|---|---|
| `lesson_preparation` | حفظ خطة التحضير لكل درس (معلم + مادة + شعبة + تاريخ) |

## شرح الأعمدة

| العمود | الترجمة | الهدف |
|---|---|---|
| `created_by` | من أعد التحضير | ربط التحضير بالمعلم |
| `subject_id` | المادة | تحديد مادة الدرس |
| `classroom_id` | الشعبة | تحديد الفصل المستهدف |
| `prep_date` | تاريخ التحضير | التخطيط الزمني |
| `lesson_title` | عنوان الدرس | تعريف سريع للموضوع |
| `objectives` | الأهداف | النتائج التعليمية المستهدفة |
| `strategies` | الاستراتيجيات | طريقة التدريس |
| `aids` | الوسائل | الأدوات المستخدمة |
| `is_approved` | حالة الاعتماد | معتمد أو بحاجة مراجعة |

## سير العملية
1. المعلم ينشئ التحضير قبل الحصة.
2. المشرف يراجع جودة التحضير.
3. التحضير يُعتمد أو يُعاد للمراجعة.
4. الإدارة تعتمد التقارير على هذه البيانات.
